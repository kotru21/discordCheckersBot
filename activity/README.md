# Корги против Биглей — 3D шашки

Визуально насыщенная игра в международные шашки (10×10) на React и Three.js. Вы играете за биглей против корги-бота: выбирайте один из режимов, перемещайте фигурки на стильной 3D-доске и сражайтесь с ИИ на базе минимакса с альфа-бета отсечением.

## Основные возможности

- 3D-сцена на React Three Fiber: объёмная доска, мягкие тени, окружение, облака и динамическая камера.
- Четыре режима игры: классика, «Безумные прыжки», «Вечеринка» с визуальными эффектами и «Турбо» с ускоренным ботом.
- Подсветка допустимых ходов и обязательные взятия по правилам международных шашек, поддержка серийных захватов.
- ИИ: минимакс с альфа-бета отсечением, настраиваемая глубина поиска и задержка по режиму.
- Индикатор производительности и авто-подстройка графики в зависимости от FPS.
- Управление камерой через OrbitControls (зум, вращение).
- Смена режима из игры (HUD), правила в модалке, экран окончания партии с рестартом.

## Технологии

- React 19 + TypeScript
- Vite 8
- Three.js, React Three Fiber, @react-three/drei
- Zustand для состояния игры и анимаций фигур
- Tailwind CSS 4

## Быстрый старт

1. Установите Node.js 22+.
2. Установите зависимости: `npm install`
3. Запустите dev-сервер: `npm run dev`
4. Откройте адрес из консоли (по умолчанию <http://localhost:5173>). Сервер стартует с `--host`, поэтому доступен и по локальной сети.

## Структура проекта

```text
src/
  components/           # UI: меню, игровой экран (GameBoard + HUD, оверлеи), модалки
  components/Board3D/ # 3D-сцена: доска, фигуры, окружение, FPS-монитор, хуки клеток
  contexts/             # GameProvider — обёртка приложения (без логики стора)
  features/animation/   # стор анимаций фигур (Zustand), хуки, интерполяция
  game/                 # чистая логика партии (matchLifecycle, подсветка клеток) и *.test.ts
  hooks/                # useGameBoardController, useBotAI, useTransientActionLock, useFullscreen
  services/             # BoardService, AIservice; подпапки move/, ai/ — правила ходов и ИИ
  shared/               # константы, типы (game.types, pieceAnimation.types и др.)
  store/                # gameStore (Zustand), matchSessionActions (рестарт + сброс анимаций)
  utils/                # логгер, gameHelpers, modeHelpers
public/
  models/               # GLB фигур и короны
  textures/             # текстуры клеток и др.
docs/                   # в т.ч. планы в docs/superpowers/plans/ (по необходимости)
```

Игровое состояние и хук **`useGame`** живут в **`src/store/gameStore.ts`**, не в `contexts/`.

## Геймплей и правила

- Вы играете за биглей, бот управляет корги. Движение только по тёмным клеткам.
- Обязательные взятия соблюдаются; если есть несколько цепочек, выбирается максимум захватов.
- При достижении противоположного края фигура становится «king» и ходит по диагонали на любое расстояние.
- Режимы:
  - Классический — стандартные правила международных шашек 10×10.
  - Безумные прыжки — допускаются длинные прыжки через всю диагональ.
  - Вечеринка — акцент на визуальные эффекты и повороты фигур.
  - Турбо — ускоренный бот (меньшая задержка, повышенная глубина поиска).
- Камера: вращение/зум через мышь; индикатор FPS и кнопки HUD (правила, режим, меню) — сверху.

## Настройки и кастомизация

- `src/shared/config/constants.ts` — глубина и задержка ИИ, размеры доски, пороги FPS, фигуры и режимы.
- `src/shared/config/menuModeItems.ts` — карточки режимов в меню и селекторе.
- `src/components/Board3D/` — освещение, окружение, облака; `EnhancedClouds` и др. влияют на плотность/вид сцены.
- `public/` — модели и текстуры; при замене файлов сохраняйте те же пути, что в коде.

## Оптимизация

- Монитор FPS переключает режимы производительности (`high` / `medium` / `low`) и передаёт данные в HUD.
- Canvas: `dpr` [1, 1.5], `powerPreference: "high-performance"`; тени и окружение зависят от режима и сцены.

## Примечание

В репозитории есть хук `useFullscreen` и поле `isFullscreen` в сторе; отдельной кнопки «полный экран» в текущем UI нет — при необходимости их можно связать вручную.
